Unit testing

Frameworks

  • xUnit: my preferred goto, very clean
  • NUnit: a good port from JUnit
  • MStest: never used it
xUnit: Applied lessons learned from NUnit JUnit: By Erich Gamma and Kent Beck (Extreme Programming Explained, Test-Driven Development & Tidy First?) MSTest: v1 was missing features (like parameterized tests) and just never looked at it again. v2 seems to be way better bUnit: Unit testing for Blazor

Test doubles

  • Dummy: to just fill in a parameter
  • Fake/shim: a actual working implementation
  • Stubs: give a specified answer
  • Spies: a stubs that also records calls made
  • Mock: have pre-programmed expected behavior
Example of a Fake is Entity Framework In memory DB Mocks: coupling tests to implementations Mock objects always use behavior verification, a stub can go either way. 🔗 Mocks (verify behaviour) Aren't Stubs (verify state) by Martin Fowler https://martinfowler.com/articles/mocksArentStubs.html

Test double tools

  • Moq: to just fill in a parameter
  • NSubstitute: a actual working implementation
  • MS Fakes: use it for fakes
Moq vs NSubstitute (the latter feel a bit cleaner to me) I mainly use NSubstitute to create Stubs and Spies.
